# Catechetical Formation — Book README

90-session catechetical formation track for young adults (18–35), built as a book inside the `base` library. Each session is one daily reading; the full course covers all 433 questions of the Catechism of St. Pius X (1912), with pastoral exposition by St. Thomas Aquinas (Naples Lent sermons, 1273) and the Roman Catechism of the Council of Trent (1566) in supporting roles.

## What this book is

A Fornasari-inspired daily formation track:

- **Image + ekphrasis** at the top of each session (Wikimedia / Met / NGA / Rijksmuseum, all PD or CC0).
- **Pius X asks** — verbatim Q&A from the 1912 catechism (the day's question).
- **St. Thomas teaches** — verbatim from his Naples sermons when topically present (~50 of 90 sessions). When Aquinas is silent, the Roman Catechism takes the teacher slot inline ("The Roman Catechism teaches").
- **Scripture cap** — one verse, DRC for EN, Pereira de Figueiredo for PT.
- **Closing** — a short authored prayer or reflection (~30 words).
- **Going Deeper** (below the rule) — the Roman Catechism's expanded treatment, shown only when Aquinas was the inline teacher above (so it doesn't duplicate).

The voice for original content (ekphrases + closings) is calibrated for young adults: direct, beautiful, intellectually respectful, reverent without being stiff. See `docs/features/catechism-pius-x/journal.md` D-07 for the audience pitch and D-15 for the slicing rationale.

Audience floor: someone curious about the Catholic faith with no prior catechesis. Audience ceiling: a soul that has read it three times and still finds the eighty-fifth session worth opening.

EN-US + PT-BR both shipped from the same source data. Italian source PDFs (Pius X) are not used in this book directly but inform the Pius X import.

The book is **self-contained**: data, original content, build pipeline, and diagnostic scripts all live inside this folder. To work on it, you only need to know `content/libraries/base/books/catechetical-formation/`.

The chapter `.md` files and `book.json` are **build outputs** — never edit them directly. Always edit the JSON inputs and re-run the build script.

## Source corpus (read-only)

Three books in the same library (`content/libraries/base/books/`):

- **`catechism-pius-x-1912/`** — Pius X 1912, all 433 Qs across 24 chapters. The doctrinal spine; verbatim quoted in every session.
- **`catechetical-instructions/`** — Aquinas's Naples Lent sermons (Collins translation, 1939). 14 Creed sermons, 12 Decalogue sermons, 9 Pater Noster + opening + short explanation, Hail Mary. Verbatim quoted (or sliced).
- **`catechism-of-trent/`** — Roman Catechism (McHugh-Callan 1923 EN / Pires Martins 1951 PT). 45 chapters. Verbatim quoted (or sliced).

This book never modifies those source books. It is a *curated anthology* — selection only, no editing of source text.

## Data schemas

### `sessions.json`

```jsonc
{
  "version": "0.5",
  "sessions": [
    {
      "id": "session-001",
      "order": 1,
      "group": "preliminary | creed | decalogue | precepts | virtues | sacraments | prayer | marian",
      "title": { "en-US": "...", "pt-BR": "..." },
      "pius_x": [
        { "chapter": "lezione-preliminare", "from": 1, "to": 13 }
      ],
      "teacher": "aquinas | trent",          // legacy advisory field; build script no longer uses it
      "aquinas": [
        // Either a chapter ID string (whole chapter):
        "creed-00-what-is-faith",
        // Or an object with optional H2 slicing:
        { "chapter": "creed-08-holy-ghost", "sections": ["@opening", "Teaching of the Nicene Creed"] }
      ],
      "trent": "creed-intro" | { "chapter": "creed-01", "sections": ["The Father", "Almighty"] } | null,
      "image_topic": "...",                  // hint for the image-curation agent
      "scripture_ref": "Genesis 1:1"         // hint; actual verse text is in content.json
    }
  ]
}
```

**Slicing rules:**
- Section anchors are matched **case-insensitive substring** against `## <heading>` lines.
- Special anchor `"@opening"` extracts the chapter content from after the H1 up to the first H2.
- Anchors are extracted in the order listed and concatenated.
- An anchor that doesn't match returns empty (silent — no error). The build script logs a warning if a session ends up with no commentary content.

## How to make changes (workflow)

**Editing prose (ekphrasis / Scripture / closing):**

1. Edit `content.json` for the relevant `session_id`.
2. Re-run the build script. Affected chapter files regenerate.

**Re-mapping a Pius X cluster:**

1. Edit `sessions.json` `pius_x` field (Q ranges).
2. Re-run the build script. Coverage audit at the top of the build will warn if any of the 433 Qs are now missing or duplicated.

**Re-slicing Aquinas or Trent commentary:**

1. Inspect the chapter's H2 structure: `grep "^## " content/libraries/base/books/catechetical-instructions/en-US/<chapter>.md`
2. Pick anchors (case-insensitive substring match). Use `@opening` for pre-H2 intro.
3. Edit `sessions.json` `aquinas` or `trent` field with `{"chapter": "...", "sections": [...]}`.
4. Re-run the build script.

**Replacing an image:**

1. Edit `images/manifest.json` for the session entry (title, artist, license, attribution, page_url, image_url, filename).
2. Replace the `images/<filename>` JPG file.
3. Re-run the build script. Markdown alt text and attribution caption update.

**Adding a new session:**

1. Append a new entry to `sessions.json` `sessions` array.
2. Append a matching entry to `content.json` `content`.
3. Add an image entry + file in `images/`.
4. Re-run the build script. The TOC in `book.json` regenerates with the new session.

## Pius X coverage audit

The build script audits coverage on every run:

```
✅ Coverage audit: all 433 Pius X questions assigned to sessions.
```

If it instead prints `⚠️ WARNING: missing Pius X Qs: [...]`, your edits to `sessions.json` left some Qs unmapped. Fix before shipping.

## Pretty-print conventions for chapter `.md` output

The build script emits chapters in this exact order:

1. `# Session NN — Title`
2. `![alt](../images/session-NN.jpg)` + attribution caption (italic)
3. `> *Ekphrasis*` (block-quote italic)
4. `## Pius X asks` + Q&A
5. `## St. Thomas teaches` (Aquinas inline) — OR — `## The Roman Catechism teaches` (Trent inline, when Aquinas is silent)
6. `> **Scripture.** *verse text* — Reference`
7. `> *Closing prayer*`
8. `---`
9. `#### Going Deeper — *Catechism of Trent*` + Trent text — *only when Aquinas was the inline teacher above*

The `---` rule is the boundary between the **daily 8–15 min read** (above) and **depth-on-demand** (below). Readers who want to keep going read past the rule; readers on a tight schedule stop at the closing prayer.

PT-BR chapters use translated section headers (`São Pio X pergunta`, `São Tomás ensina`, `O Catecismo Romano ensina`, `Aprofundamento`).

## Scripts (inside `scripts/`)

- **`build.py`** — main build pipeline. Run after any edit to `sessions.json`, `content.json`, or `images/manifest.json`. Emits 180 chapter `.md` files + `book.json`.
- **`slice.py`** — one-time migration that applied the per-cluster H2 slicing plan to `sessions.json`. Re-runnable; the `PLAN` dict inside the script is the source of truth for slicing decisions. Adjust `PLAN` to revise slicing.
- **`audit.py`** — diagnostic. Prints each session's order/title/first-and-last Pius X Q/Aquinas slice/Trent slice. Use to scan for title-content mismatches before shipping.
